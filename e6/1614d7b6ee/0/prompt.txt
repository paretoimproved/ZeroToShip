# Fix Generate Spec ‚Äî Visibility, Incentive, UX, Notifications

## Context
The "Generate Spec" feature lets users turn an idea brief into a developer-ready spec (user stories, DB schema, API routes, CLAUDE.md). The user reported 4 problems:
1. The button is hidden below the fold on idea pages and completely missing from the archive modal
2. Free users have no preview/demo of what a spec contains ‚Äî no incentive to upgrade
3. After clicking generate, errors cause a page refresh with no persistent feedback
4. No notification or link when the spec completes ‚Äî user doesn't know where to find it

## Plan

### 1. Backend: Add `GET /specs/usage` endpoint
**File:** `src/api/routes/specs.ts`

Add a new route **before** the `/:id` route (so Fastify doesn't parse "usage" as a UUID param):

```
GET /api/v1/specs/usage ‚Üí { used: number, limit: number, tier: string }
```

- Import `getMonthlySpecCount` from `../services/ideas` and `getMonthlySpecLimit` from `../config/tiers`
- Requires `requireAuth` middleware
- Returns the user's current month spec count, their tier limit, and tier name

### 5. Frontend: Create `GenerateSpecCta` component
**New file:** `web/components/GenerateSpecCta.tsx`

Self-contained component that handles all spec generation UX:

**Props:**
```typescript
interface GenerateSpecCtaProps {
  ideaId: string;
  ideaName: string;
  isAuthenticated: boolean;
  userTier?: string;
  specUsage?: { used: number; limit: number };
  onSpecGenerated?: (spec: AgentSpec, generationId: string) => void;
}
```

**Rendering logic (3 states):**

1. **Not authenticated** ‚Üí "Sign up to generate specs" with link to `/signup`
2. **Quota exhausted** (`specUsage.used >= specUsage.limit`) ‚Üí Upgrade teaser:
   - Left: "You've used all X generations this month" + description of what specs include (user stories, DB schemas, API routes, CLAUDE.md instructions)
   - Right: "Upgrade to Pro" button linking to `/pricing`
3. **Quota available** ‚Üí Generate button + quota indicator:
   - Button: "Generate Agent Spec" with code icon, spinner when loading
   - Below button: "X of Y remaining this month" in muted text

**Behavior on click:**
- Calls `api.generateSpec(ideaId)`
- On success: fires `toast.push()` with variant "success", title "Spec generated!", and a JSX description containing a `<Link>` to `/specs/{generationId}`. Duration 8s (longer since it has an actionable link). Calls `onSpecGenerated` callback.
- On error: fires `toast.error()` with the error message
- Loading state: button disabled, shows spinner + "Generating..."
