# IdeaForge

A social signal scraper for identifying emerging startup ideas.

## Project Structure

- **Code**: `~/Projects/ideaforge/` (this directory)
- **Documentation**: `~/DevVault/01-Projects/ideaforge/`

## Key Documentation

All planning docs live in DevVault:

- `Context.md` - Current state, blockers, and progress
- `Feature-Plan.md` - Implementation roadmap
- `Agent-Instructions.md` - Multi-agent coordination
- `Daily/` - Daily work logs (auto-generated by git-to-daily)

## Development

```bash
npm install
npm test
npm run build
```

## Environment

Requires `GIT_TO_DAILY_VAULT` environment variable for automatic daily logs.

## Agent Rules

When completing any agent task, follow the **Agent Completion Protocol** (see `Agent-Instructions.md` for full details):

### 1. Verify Before Commit
```bash
npm run build && npm test  # Both must pass
```

### 2. Commit with Agent Tag
```bash
git add <specific files>
git commit -m "<type>(agent-<N>): <short description>

<details>

Phase <X>: <Phase Name>

Co-Authored-By: Claude <model> <noreply@anthropic.com>"
```

**Types**: `feat`, `fix`, `test`, `docs`, `refactor`, `chore`

### 3. Push to Remote
```bash
git push origin main
```

### 4. Update Documentation
- **Context.md** (`~/DevVault/01-Projects/ideaforge/Context.md`):
  - Status update in Agent Communication Log
  - Test count changes
  - Blockers resolved

- **Agent-Instructions.md** (`~/DevVault/01-Projects/ideaforge/Agent-Instructions.md`):
  - Mark agent status as ✅ COMPLETE in Quick Reference table
  - Note any execution flow changes
