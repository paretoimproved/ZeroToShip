# Demo Assets

Use this folder for short artifacts that support interview walkthroughs.

## Suggested Structure

1. `demos/2026-02-13-mvp-walkthrough/`
2. `demos/2026-02-13-mvp-walkthrough/script.md`
3. `demos/2026-02-13-mvp-walkthrough/checklist.md`
4. `demos/2026-02-13-mvp-walkthrough/assets/`

## 5-Minute Script Template

```md
Title:
Audience:
Goal:

1. Problem and user pain (45s)
2. Agentic workflow architecture (75s)
3. Live path: trigger -> pipeline run -> admin visibility -> output (120s)
4. Reliability and hardening evidence (60s)
5. Current migration plan and next milestones (60s)

Key metrics to mention:
- Delivery throughput (from reviews/career/impact-scoreboard.md)
- Quality/safety controls (tests, CI, security hardening)
- Current KPI gaps and what is being instrumented next
```

## Capture Checklist

1. Record one demo from a clean local start.
2. Capture one fallback/edge case path, not only the happy path.
3. Include one proof artifact for reliability (CI/test output or diagnostics screen).
4. Update corresponding entries in `reviews/career/experiment-log.md` and `reviews/career/weekly-build-log.md`.
