# Career Demo Documentation Pack

Last updated: 2026-02-13

## Purpose

Use this folder as objective evidence for leadership and IC interviews in agentic AI teams:

1. Show shipping pace.
2. Show decision quality under uncertainty.
3. Show reliability and operational discipline.
4. Show product judgment and iteration loops.

## Audience Map

1. Recruiters: start with `impact-scoreboard.md` and `weekly-build-log.md`.
2. Hiring managers: read `decision-log.md` and `experiment-log.md`.
3. Technical interviewers: use commit hashes in each log to drill into implementation.

## Files

1. `experiment-log.md`: hypothesis -> change -> outcome -> decision loop.
2. `decision-log.md`: leadership tradeoffs and why each decision was made.
3. `impact-scoreboard.md`: delivery and quality metrics from repository history.
4. `weekly-build-log.md`: weekly narrative of what shipped, what broke, and what changed.

## Update Cadence

1. After each meaningful session: add at least one entry to `experiment-log.md`.
2. When a material tradeoff is made: append to `decision-log.md`.
3. Weekly: refresh `impact-scoreboard.md` and add one `weekly-build-log.md` entry.
4. After each demo-worthy milestone: add assets in `/demos` and reference them in the logs.

## Evidence Standard

1. Prefer measurable signals over adjectives.
2. Do not claim numeric outcomes without a source.
3. Attach commit hashes, test commands, or artifact links for each claim.
4. Note open risks and unresolved questions explicitly.
