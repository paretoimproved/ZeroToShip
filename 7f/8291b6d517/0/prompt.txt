# ZeroToShip Audit Roadmap — P0 through P3

## Context
7 expert agents audited the entire codebase and produced 138 actionable findings across security, code quality, accessibility, product, growth, analytics, and finance. This roadmap synthesizes and prioritizes all findings into 4 tiers based on severity, user impact, and dependencies.

**Prioritization criteria:**
- **P0 (Ship-blocking):** Security vulnerabilities, data loss risks, things that would embarrass us in production
- **P1 (This sprint):** High-impact conversion/activation fixes, critical UX gaps, code quality that affects reliability
- **P2 (Next sprint):** Feature gaps for PMF, growth experiments, analytics instrumentation
- **P3 (Backlog):** Polish, optimization, nice-to-haves, longer-term features

---

## P0 — Ship-Blocking (Do Now)

### P0-1: Fix unsubscribe token IDOR vulnerability
- **Why:** Unsubscribe uses predictable user UUID — anyone can mass-unsubscribe users by iterating UUIDs
- **Source:** Security C-1
- **Files:** `src/delivery/email.ts:180`, `src/api/routes/user.ts:47-73`
- **Fix:** Generate cryptographic random tokens, store in DB, validate on unsubscribe
- **Effort:** Medium

### P0-2: Fix Docker secrets leak
- **Why:** `.env` with Stripe/Supabase keys gets copied into Docker image via `COPY . .`
- **Source:** Security C-2
- **Files:** `Dockerfile:10`, `.dockerignore`
- **Fix:** Add `.env`, `.env.*`, `web/`, `tests/`, `docs/` to `.dockerignore`
- **Effort:** Small

### P0-3: Add Zod validation to admin routes (eliminate `as any`)
- **Why:** JSONB `phaseResults` accessed with `as any` — malformed data crashes the server
- **Source:** Staff Engineer C-1, Security H-3, UX C-13
- **Files:** `src/api/routes/admin.ts:104-116, 315, 322-325, 347, 353, 362, 412, 419-420`
- **Fix:** Add Zod schemas for all admin request bodies and JSONB field access
- **Effort:** Medium

### P0-4: Fix `requireAuth` middleware control flow
- **Why:** Missing `return` after `reply.send()` allows unauthenticated requests to continue
- **Source:** Security H-2, Staff Engineer C-2
- **Files:** `src/api/middleware/auth.ts:237-249`
- **Fix:** Add `return` after 401 response
- **Effort:** Small

### P0-5: Provision Redis on Railway
- **Why:** In-memory fallback loses rate-limit state, cache, and session data on restart
- **Source:** Product, CFO
- **Fix:** Provision Redis add-on on Railway, verify `src/config/redis.ts` connects
- **Effort:** Small (ops)

### P0-6: Set up monitoring (UptimeRobot + Sentry)
- **Why:** No production alerting — outages and errors are invisible
- **Source:** Product
- **Fix:** Configure UptimeRobot for health endpoint, Sentry for error tracking
- **Effort:** Small (ops)
